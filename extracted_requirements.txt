FieldSense

Field Operations Visibility and Asset Condition Monitoring System



PART A: PROJECT REQUIREMENTS DOCUMENT

1. Functional Requirements

1. Asset Management:• Add, edit, and delete assets• Store asset details (ID, type, location, installation date, status)• View asset list and history2. Issue Reporting System:• Manual issue reporting• Issue fields (Asset ID, Issue type, Description, Priority, Timestamp)• Update issue status (Open/Assigned/Resolved)• Maintain logs3. Technician Assignment:• Admin assigns technician• Technician updates repair progress• Store resolution notes4. IoT Monitoring Integration:• Sensors send periodic updates• Detect power failure, temperature breach, device offline• Auto-generate issue tickets5. Dashboard:• Total assets• Active issues• Resolved issues• Fault distribution• Recent alerts

2. Non-Functional Requirements

Reliability:• Log failures and sensor events• Handle invalid input safelyPerformance:• Dashboard loads within 2 seconds• Issue creation under 1 secondSecurity:• Role-based login• Password hashing• Secure API endpointsUsability:• Mobile-friendly interface• Simple reporting workflowScalability:• Support increasing number of assets and issues

3. System Architecture Requirements

Frontend: Web interface using HTML/CSS/JavaScript or ReactBackend: REST API using Node.js / Python / PHPDatabase: MySQL or MongoDBCore Tables:UsersAssetsIssuesTechniciansSensorLogs

4. IoT Requirements

Sensors:• Temperature sensor• Power detection sensor• Microcontroller (ESP8266 / ESP32 / Arduino)IoT Workflow:Sensor → Microcontroller → HTTP Request → Backend API → Database → Dashboard Alert

5. Deployment Requirements

• Backend server (local or cloud)• Database server• Git version control• Basic logging enabled

Project Goal

To improve visibility of distributed field assets and reduce manual monitoring by combining issue tracking with optional IoT-based fault detection.

PART B: FEASIBILITY REPORT

1. Introduction

This report evaluates the feasibility of developing FieldSense, a centralized platform designed to monitor distributed field assets using IoT sensor data and manage service workflows efficiently. The system integrates asset lifecycle management, issue tracking, technician assignment, IoT-based anomaly detection, and real-time dashboards.

2. Objective of the Project

• Develop a centralized asset monitoring platform.• Enable proactive fault detection using IoT sensors.• Automate issue ticket generation.• Streamline technician assignments.• Provide real-time operational dashboards.• Reduce downtime and maintenance costs.

3. Scope of the Project

Included:• Asset registration and tracking• Manual issue reporting• Technician assignment• Basic IoT monitoring (temperature, power, device status)• Role-based authentication• Dashboard analytics• Small-scale industrial deployment

4. Problem Statement

Organizations managing distributed equipment (Amul, Hershey, Milky Mist etc) often face delayed fault detection, high downtime, and inefficient service workflows. FieldSense aims to solve these challenges using IoT-based monitoring and structured issue management.

5. Technical Feasibility
